# PeekView

> Agent writes, humans view, agents can also read. Publish agent output, get a link — rendered in browser, readable via /raw endpoint.

PeekView is a self-hosted rendering service for AI agent output. Agents publish via MCP, CLI, or REST API. Humans open the link in a browser. Other agents read the raw content back via GET /api/v1/entries/{slug}/raw.

Two packages: **peekview** (Python, pipx install) provides the backend + frontend + CLI. **@peekview/mcp-server** (Node.js, npm install) provides the MCP bridge.

## API

- [Agent read path](https://github.com/randomgitsrc/peekview/blob/main/README.md): GET /api/v1/entries/{slug}/raw — structured JSON, public entries no auth, private entries need API key
- **Content Negotiation**: GET /{slug} with `Accept: application/json` returns JSON (same as /raw); `Accept: text/html` or default returns HTML. JSON only when `application/json` is acceptable AND `text/html` is NOT. `*/*` returns HTML. Matches GitHub behavior.
- **HTML self-description**: HTML responses for valid slugs include `<link rel="alternate" type="application/json" href="/api/v1/entries/{slug}/raw" />` in `<head>` and `Link` HTTP header, enabling discovery without prior knowledge of /raw.
