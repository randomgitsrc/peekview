## 新增内容评审

### `validation-plan.md`

**设计评估：优秀。**

- 问题意识正确：v3 是文档协议不是代码框架，必须验证 LLM 能否自觉执行
- 验证顺序正确：先方法 B（general + prompt 注入，v3 主力路径），再方法 A（自定义角色，锦上添花）
- Phase 3 的「大文件测试」（500 行）直接验证上下文隔离，是 v3 价值的最终证明
- 判定矩阵覆盖 4 种场景（理想/可用/需调整/不可用），不假设一定成功
- 「不要跳过渐进过程」的提醒正确对齐 loop-orchestration.md 的档位 A→B→C

**1 个小问题**：Phase 2.2 用 `opencode agent list` 验证自定义角色，但 OpenCode 当前版本可能没有这个子命令。验证方案应标注「若命令不存在，跳过 Phase 2 的自动判定，改用手动 `@test-echo` 方式测试」。
