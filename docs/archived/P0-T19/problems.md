# Task 19: 问题定义 - 完整测试体系建设

## 任务概述

建立 PeekView 项目的完整测试体系，覆盖前端显示、交互功能、桌面端和移动端。

---

## 问题清单

### P1: 缺乏 E2E 测试覆盖
**现象**: 前端单元测试通过但无法验证实际浏览器渲染效果  
**期望**: 有自动化 E2E 测试验证关键用户流程  
**验收标准**:
- [ ] 可连接 Chrome CDP 执行测试
- [ ] 覆盖 EntryListView 和 EntryDetailView
- [ ] 截图验证视觉正确性

### P2: 缺乏交互功能测试
**现象**: 按钮点击、复制、下载等交互行为未测试  
**期望**: 自动验证所有交互功能正常工作  
**验收标准**:
- [ ] Copy 按钮复制内容到剪贴板
- [ ] Download 按钮触发下载
- [ ] 主题切换正常工作
- [ ] Wrap 按钮切换代码换行

### P3: 缺乏桌面端测试
**现象**: 桌面端布局和显示未系统验证  
**期望**: 桌面端 (1920x1080) 所有页面正常显示  
**验收标准**:
- [ ] EntryListView 桌面端布局正确
- [ ] EntryDetailView 桌面端布局正确
- [ ] CodeViewer 在桌面端显示正常
- [ ] 底部工具栏不遮挡内容

### P4: 缺乏移动端测试
**现象**: 移动端适配未系统验证  
**期望**: 移动端 (375x667) 所有页面正常显示  
**验收标准**:
- [ ] EntryListView 移动端布局正确
- [ ] EntryDetailView 移动端布局正确
- [ ] 底部工具栏在移动端显示
- [ ] 触摸操作正常

### P5: 缺乏视觉回归测试
**现象**: UI 变更可能导致样式回归  
**期望**: 关键页面有基线截图对比  
**验收标准**:
- [ ] 首页截图基线
- [ ] 详情页截图基线
- [ ] 深色/浅色主题截图基线

---

## 测试策略 (P0.2)

| 问题 | 测试类型 | 工具 | 说明 |
|------|----------|------|------|
| P1 | E2E 测试 | Playwright + CDP | 连接现有 Chrome 实例 |
| P2 | E2E 测试 | Playwright | 模拟用户交互 |
| P3 | E2E 截图测试 | Playwright | 1920x1080 viewport |
| P4 | E2E 截图测试 | Playwright | 375x667 viewport (iPhone SE) |
| P5 | 视觉回归 | Playwright screenshot | 与基线对比 |
