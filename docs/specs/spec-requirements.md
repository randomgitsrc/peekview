## 1. 项目概述

**Peek** 是一个轻量级代码与文档格式化展示服务。

核心定位：**Agent 产出 → Peek 格式化 → 人类友好查看**

Agent（AI 智能体）生成代码片段、代码文件、Markdown 文档等内容后，通过 Peek 的 API 或 CLI 创建条目，获得一个可分享的链接。用户在浏览器中打开链接，即可查看格式化、高亮、美观的内容页面。

---

## 2. 核心用户与场景

### 2.1 主要用户

👤 **AI Agent** — 通过 API / CLI / MCP 调用，创建内容条目，获取链接  
👤 **人类用户** — 在浏览器中查看内容页面，管理索引

### 2.2 使用场景

- **Agent 产出展示** — AI 生成代码/文档后，生成链接供人类查看
- **团队分享** — 将格式化内容链接分享给他人
- **个人知识库** — 长期存储和检索代码片段与文档

## 4. 用户故事与验收标准

### US-01：Agent 创建多文件代码展示

> 作为 Agent，我想创建一个包含多文件的代码展示条目，以便人类用户可以通过链接查看格式化后的代码。

**验收标准：**
- 支持内容直传和本地路径引用两种方式
- 支持传入目录自动递归扫描
- 多文件保留目录结构，前端渲染为目录树
- 返回可访问的 URL

### US-02：Agent 指定自定义 Slug

> 作为 Agent，我想指定自定义 URL slug，以便分享的链接具有可读性和语义。

**验收标准：**
- slug 格式限制为 `[a-z0-9_-]`，最长 64 字符
- 不传 slug 时自动生成短随机码
- slug 冲突时自动追加 `-N` 后缀
- API 响应返回实际 URL（Agent 无需猜测后缀）

### US-03：人类用户查看语法高亮代码

> 作为人类用户，我想在浏览器中查看语法高亮的代码，以便快速理解代码内容，无需下载和打开 IDE。

**验收标准：**
- 代码块有行号
- 支持自动换行切换
- 语言根据文件扩展名自动检测
- 高亮效果与 VSCode 一致
- 不支持的语言回退到纯文本展示

### US-04：人类用户搜索和过滤条目

> 作为人类用户，我想在索引页搜索和过滤条目，以便快速找到我需要的内容。

**验收标准：**
- 支持关键词搜索（概述 + 文件名 + 文件内容）
- 支持按标签过滤
- 支持搜索 + 标签 + 分页组合
- 搜索范围：概述文本、文件名、文本文件内容

### US-05：Agent 设置条目过期时间

> 作为 Agent，我想设置条目过期时间，以便临时展示的内容能自动清理。

**验收标准：**
- 支持过期时间格式：`7d`, `30d`, `1h` 等
- 过期后自动删除条目及关联文件
- 无过期时间的条目永久保留

### US-06：人类用户复制代码

> 作为人类用户，我想复制代码片段到剪贴板，以便在 IDE 中直接使用。

**验收标准：**
- 代码文件右上角有一键复制按钮
- Markdown 内代码块右上角有一键复制按钮
- 复制成功有视觉反馈

### US-07：人类用户移动端查看

> 作为人类用户，我想在手机上查看代码内容，以便在移动场景下也能访问。

**验收标准：**
- 内容区自适应屏幕宽度
- 桌面端三栏布局在移动端切换为单栏
- **底部工具栏**：固定显示 Wrap、Copy、Download 按钮
  - 代码文件显示 Wrap 按钮，Markdown 隐藏
  - Markdown 有标题时显示 TOC 按钮，代码文件隐藏
  - 多文件条目左侧显示汉堡按钮 + "N files" 徽章
  - 单文件直接显示文件名（无汉堡按钮）
- **文件抽屉**：底部弹出，显示完整文件树
- **TOC 抽屉**：底部弹出，显示 Markdown 大纲
- 代码区支持左右滑动
- 代码复制时不复制行号

### US-08：Agent 通过 MCP 调用

> 作为 Agent，我想通过 MCP 协议调用 Peek，以便无需了解 REST API 细节即可集成。

**验收标准：**
- 提供 MCP Server，封装核心 API 为工具
- 工具参数有 schema 定义
- 配置到 Hermes MCP servers 即可自动发现

### US-09：人类用户下载文件

> 作为人类用户，我想下载条目中的文件或打包下载，以便离线查看或编辑代码。

**验收标准：**
- 单文件下载
- 多文件打包为 zip 下载
- 二进制文件提供下载链接

### US-10：人类用户切换主题

> 作为人类用户，我想在深色/浅色模式下查看内容，以便适配我的阅读习惯和环境光线。

**验收标准：**
- 自动跟随浏览器 `prefers-color-scheme`
- 支持手动切换
- 切换后记忆偏好

## 4. 功能需求

### 4.1 后端服务

#### 4.1.1 API 接口

- **创建条目** — 传入概述 + 文件（本地路径或内容直传），返回页面 URL
- **更新条目** — 更新已有条目的内容（概述、新增文件、删除文件）
- **删除条目** — 删除条目及其关联文件
- **条目列表** — 返回所有条目（支持分页、搜索、标签过滤）
- **条目详情** — 返回单个条目的完整信息
- **条目状态管理** — 通过更新 status 字段实现存档/发布
- **文件下载** — 单文件下载、打包下载（zip）
- **健康检查** — `/health` 端点

#### 4.1.2 文件上传方式

| 方式 | 说明 |
|------|------|
| 本地路径引用 | Agent 传服务器上的文件路径，后端**读取并复制**到数据目录，**不删除原文件** |
| 内容直传 | Agent 把文件内容直接放在请求体里（text 或 base64） |
| 目录上传 | Agent 传目录路径，后端**递归扫描**目录下所有文件，保留目录结构 |

三种方式均支持，可混合使用。

#### 4.1.3 local_path 安全机制

- **黑名单目录** — 禁止读取以下敏感路径：
  - `~/.ssh/`
  - `~/.gnupg/`
  - `~/.env`、`*.env` 文件
  - `/etc/shadow`、`/etc/passwd`
  - 其他配置中指定的黑名单路径
- **复制而非移动** — 读取文件后复制到 peek 数据目录，绝不删除/移动原文件
- **路径遍历防护** — 解析 `resolve()` 后的路径，拒绝包含 `..` 的路径
- **符号链接** — 不解析符号链接，避免绕过黑名单

#### 4.1.4 资源限制

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 单文件大小 | 10 MB | 内容直传时 1 MB，local_path 时 10 MB |
| 条目文件数 | 50 | 单条目最多文件数 |
| 条目总大小 | 100 MB | 单条目所有文件总大小 |
| slug 长度 | 64 字符 | 自定义 slug 最长 |
| 概述长度 | 500 字符 | summary 最长 |
| 每页条目数 | 20 | 列表分页，最大 100 |

超出限制时 API 返回 413 错误，明确提示哪个限制被触发。

#### 4.1.5 API 错误响应格式

所有 API 错误使用统一格式：

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "人类可读的错误描述",
    "details": {}
  }
}
```

**错误码清单：**

| HTTP 状态码 | 错误码 | 说明 |
|-------------|--------|------|
| 400 | `VALIDATION_ERROR` | 参数校验失败（缺字段、格式错误） |
| 400 | `INVALID_SLUG` | slug 格式不合法 |
| 403 | `FORBIDDEN_PATH` | local_path 在黑名单中 |
| 404 | `NOT_FOUND` | 资源不存在 |
| 404 | `FILE_NOT_FOUND` | local_path 指向的文件不存在 |
| 413 | `PAYLOAD_TOO_LARGE` | 超出资源限制 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

#### 4.1.6 CLI 命令

```
peek serve                          # 启动服务
peek create "概述" --files a.py b.md --dir ./src [--slug my-doc] [--tags tag1,tag2] [--expires 7d]
peek list [--search 关键词] [--tags tag1] [--page 1]
peek get <id>
peek update <id> [--summary "新概述"] [--add-files c.py] [--add-dir ./lib] [--remove-files b.md]
peek delete <id>
```

> 注：archive/publish 功能通过 `peek update <id> --status archived` 实现，不单独设命令。

#### 4.1.7 MCP Server

- 将 API 封装为 MCP 工具，供 AI Agent 原生调用
- 工具列表：`peek_create`, `peek_list`, `peek_get`, `peek_update`, `peek_delete`
- 配置到 Hermes 的 MCP servers 即可自动发现

#### 4.1.8 URL 与 Slug

- 条目 URL 使用 **slug** 作为标识（如 `/view/my-design-doc`）
- 不传 slug → 自动生成 6 位短随机码（如 `xK9m2p`），不冲突
- 传 slug → 没冲突直接使用，冲突时自动追加 `-N` 后缀
- slug 格式：`[a-z0-9_-]`，1-64 字符
- API 响应返回实际 URL（Agent 不需要猜测后缀）
- 域名可配置，未配置时返回本地地址

#### 4.1.9 认证

- 当前版本：无认证（本地使用）
- 预留扩展：未来支持 API Key / Token 认证
- 默认监听 `127.0.0.1`，`0.0.0.0` 需显式配置

### 4.2 前端展示

#### 4.2.1 代码文件展示

- 支持 100+ 语言语法高亮（Shiki）
- 支持格式：cpp, h, cs, java, html, xml, yaml, json, bash, makefile, txt 等
- 语言自动检测（根据文件扩展名）
- 不支持的语言回退到纯文本展示
- 支持中文编码（UTF-8）
- 长行内容：支持自动换行切换（开关按钮）
- 一键复制代码（右上角按钮）
- 文件行数统计（如"128 行"）

#### 4.2.2 Markdown 文件展示

- 标准 Markdown 渲染（markdown-it + 插件）
- 目录标题大纲（TOC），支持点击跳转
- 代码片段高亮 + 右上角一键复制按钮
- 表格合理显示，超出宽度支持左右拖动
- 支持图片内联显示
- 支持 Mermaid 图表渲染
- Mermaid 渲染失败时降级显示原始代码块
- 正文适应显示宽度
- **XSS 防护** — 禁止渲染原始 HTML 标签，使用 `sanitize-html` 过滤恶意内容

#### 4.2.3 多文件展示

- 支持目录结构树形显示（保留相对路径信息）
- 单文件/少量文件自动扁平显示
- 有路径信息时渲染成树，无路径信息时渲染成扁平列表
- 点击文件名切换查看内容
- 目录树与内容区联动

#### 4.2.4 二进制文件

- 不做内容渲染
- 图片文件（.png/.jpg/.svg/.gif/.webp）内联显示
- 其他二进制文件提供下载链接和文件大小信息

#### 4.2.5 空文件处理

- 0 字节文件显示"空文件"提示
- 不阻止创建

#### 4.2.6 通用功能

- **暗色/亮色模式** — 根据浏览器 `prefers-color-scheme` 自动切换，支持手动切换
- **响应式布局** — 自适应桌面浏览器与手机终端
  - 手机端：目录树和大纲折叠为菜单/抽屉
- **一键复制** — 单文件复制
- **下载** — 单文件下载
- **打包下载** — 多文件打包为 zip 下载
- **URL 定位** — 支持 `?file=main.py` 或 `#L5-L10` 直接定位到指定文件/行

### 4.3 索引与管理页面

#### 4.3.1 索引页

- 显示所有条目列表
- 分页显示（默认每页 20 条，最大 100）
- 按创建时间倒序排列
- 搜索功能（关键词搜索概述、文件名、文件内容）
- 按标签过滤
- 点击条目跳转详情页

#### 4.3.2 管理操作

- 删除条目
- 更新条目状态（active / archived / published）

#### 4.3.3 状态语义

| 状态 | 含义 | 索引页可见 |
|------|------|-----------|
| active | 正常条目 | ✅ |
| archived | 已存档，不再活跃 | ✅（默认隐藏，可切换显示） |
| published | 已发布，对外可见 | ✅ |

#### 4.3.4 标签

- 条目可打标签
- 标签由 Agent 维护（API 传入）
- 索引页支持按标签筛选

---

### 4.4 数据生命周期

- 条目默认永久保留
- 可选设置过期时间（创建时指定，如 `7d`, `30d`）
- 过期后自动删除条目及关联文件
- 后端启动时检查过期条目并清理
- 运行时按配置间隔定期检查（可关闭）

## 5. 非功能需求

### 5.1 性能

- API 响应时间 < 200ms（常规请求）
- 页面加载时间 < 1s（不含网络延迟）
- 代码高亮首屏渲染 < 500ms

### 5.2 部署

- 本地开发：`peek serve` 一键启动
- Docker 部署：提供 Dockerfile
- 远程服务器：支持域名配置

### 5.3 扩展性

- 数据模型预留多用户字段（`user_id`，当前固定 `'default'`）
- 认证模块预留接口
- MCP Server 可独立扩展
- 文件存储抽象层预留（当前本地文件系统，未来可切换 S3）
- 搜索抽象层预留（当前 FTS5，未来可切换 Elasticsearch）

### 5.4 可靠性

- SQLite 数据库（WAL 模式）+ 本地文件，无外部依赖
- 备份：复制数据目录即可
- 创建条目使用事务保证一致性

### 5.5 安全

- local_path 黑名单机制
- 路径遍历防护
- Markdown XSS 防护
- 默认监听 127.0.0.1

## 8. 术语

| 术语 | 含义 |
|------|------|
| 条目 (Entry) | 一个展示单元，包含概述 + 一个或多个文件 |
| Slug | 条目的 URL 标识（自动生成或自定义） |
| Agent | AI 智能体（如 Hermes），通过 API/CLI/MCP 调用 Peek |
